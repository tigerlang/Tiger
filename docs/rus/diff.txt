Tiger v0.3 (18.05.2026, нестабильный/неприменимый):
Добавлено:
- Опциональная статическая типизация (x: int = 5), добавлен Type Checker под статику. Статическая типизация не обязательна, но желательна при масштабировании программы на Tiger

- Исправлен баг с глобальными переменные.

- Компилятор и CLI название сменено с 'tigrc' на 'tiger'

- Поддержка словарей (dict): var config = {"key": value}. Значения могут быть любыми типами (int, float, string, bool, list). Транслируется в std::map<std::string, std::any>. Доступ по ключу: config["key"]

- Исправлен генератор C++ для словарей: теперь использует инициализацию списком {}

- Исправлен флаг -municode в g++ (убран, не поддерживается MinGW 6.3)

Tiger v0.4.1 (23.05.2026, нестабильный/неприменимый):
Добавлено:
- for-циклы: синтаксис for (item in collection) для перебора списков, строк и других итерируемых объектов
- Строковые методы:
  * str.trim(s) — удаление пробелов в начале и конце строки
  * str.split(s) — разделение строки на список подстрок (по пробелам)
  * str.startswith(s, prefix) — проверка начала строки с префикса
  * str.endswith(s, suffix) — проверка окончания строки суффиксом

- Теперь скобки вокруг выражений у while, if/elif/else и for НЕ ОБЯЗАТЕЛЬНЫ:
* Раньше:
if (a == 5) {...} # Нельзя убрать круглые скобки
* В 0.4.2:
if a == 5 {...} # Это поддерживается!
- Теперь люди привыкшие ставить круглые скобки могут это делать, а те кто привыкли к пайтону - делают как хотят в этом плане.

Tiger v0.4.2 (25.05.2026):
Добавлено:

- Исправлен баг со словарями: теперь принимаются любые типы данных (int, float, string, bool, list), а не только числа (double). Используется std::any для хранения значений.
 - Новая встроенная библиотека files:
   - files.create_dir(path) — создание директории
   - files.create_file(path) — создание файла
   - files.remove_dir(path) — удаление директории
   - files.remove_file(path) — удаление файла
   - files.list_dir(path) — возвращает список имён файлов и папок
   - files.read(path) — чтение содержимого файла
   - files.write(path, content) — запись данных в файл
