## 1. Что такое Tiger

Tiger — это компилируемый язык программирования с синтаксисом, похожим на Python и JavaScript. Исходный код `.tgr` транспилируется в C++, а затем компилируется `g++` в нативный исполняемый файл `.exe` (Windows) или бинарник (Linux/Mac).

**Философия языка:**
- Простой, читаемый синтаксис без лишних символов
- Статическая компиляция — результат работает без интерпретатора
- Типы выводятся автоматически — не обязательно писать `int`, `string` и т.д., хотя можно для дебагга.
- Всё, что можно сделать в C++, можно выразить в Tiger

## 2. Запуск и компиляция

### Установка

Для работы нужен Python 3.9+ и компилятор g++ (MinGW-w64 на Windows).

```
python tiger.py <файл.tgr> [опции]
```

### Опции командной строки

| Опция | Описание |
|-------|----------|
| `-o <путь>` | Имя выходного исполняемого файла |
| `--emit-cpp` | Остановиться после генерации `.cpp`, не компилировать |
| `--cpp-out <f>` | Указать имя для `.cpp` файла |
| `-I <dir>` | Добавить папку для поиска `.tgr` модулей |
| `-v` / `--verbose` | Подробный вывод о каждом шаге |

### Примеры вызова

```bash
# Скомпилировать hello.tgr в hello.exe
python tiger.py hello.tgr

# Указать имя выходного файла
python tiger.py hello.tgr -o myapp.exe

# Только сгенерировать C++, не компилировать
python tiger.py hello.tgr --emit-cpp

# Посмотреть все шаги
python tiger.py hello.tgr -v

# Добавить папку с модулями
python tiger.py main.tgr -I ./libs
```

## 3. Структура программы

Tiger поддерживает два стиля написания программ.

### Стиль 1: С функцией main() — рекомендуется

```tiger
# Вспомогательные функции объявляются до main
greet(name) {
    print("Привет, " + name + "!")
}

main() {
    greet("мир")
}
```

Функция `main()` — точка входа. Всё, что в ней, выполняется при запуске.

### Стиль 2: Скриптовый — без main()

```tiger
var x = 10
var y = 20
print("Сумма: " + str.to_str(x + y))
```

В скриптовом стиле верхнеуровневые операторы (кроме `var` и функций) выполняются последовательно в сгенерированном `main()`. Однако **переменные на верхнем уровне становятся глобальными** и доступны из функций.

### Порядок секций в файле

```
[глобальные переменные]   ← var снаружи функций
[функции]                 ← любые функции кроме main
[main()]                  ← точка входа (необязательна)
```

## 4. Комментарии

Tiger поддерживает только однострочные комментарии:

```tiger
# Это комментарий

var x = 5  # комментарий в конце строки

# Многострочный "блок" — просто несколько строк с #
# Каждая строка начинается с решётки
# Многострочных /* */ комментариев нет
```

Также работает `//` (синтаксис C++):

```tiger
// Это тоже комментарий — работает так же как #
var y = 10  // конец строки
```

## 5. Типы данных

Tiger поддерживает статическую типизацию. Типы можно указывать явно или опускать для автоматического вывода.

### Основные типы

| Тип | Синтаксис | Описание |
|-----|-----------|----------|
| `int` | `var x: int = 42` | Целые числа |
| `float` | `var y: float = 3.14` | Дробные числа (в C++: `double`) |
| `string` | `var s: string = "hello"` | Строки |
| `bool` | `var flag: bool = true` | Булевы значения |
| `null` | `var n: null = null` | Отсутствие значения |
| `list` | `var lst: list = [1, 2, 3]` | Списки (по умолчанию `vector<double>`) |
| `list[T]` | `var nums: list[int]` | Список с типом элементов |
| `dict` | `var cfg: dict = {"key": 1}` | Словари (`map<string, any>` в C++) |

### Явное указание типов (рекомендуется)

```tiger
var age: int = 25
var price: float = 99.99
var name: string = "Tiger"
var active: bool = true

var numbers: list = [1, 2, 3, 4, 5]
var empty_list: list = []
```

### Автоматический вывод типов (без указания)

```tiger
var x = 42        # int - автоматически
var y = 3.14      # float - автоматически  
var s = "hello"   # string - автоматически
```

### Целые числа (int)

```tiger
var a = 42
var b = -7
var c = 1000000
```

### Дробные числа (float / double)

```tiger
var pi = 3.14159
var temperature = -12.5
var rate = 0.01
```

> **Нюанс:** Число `1.` (с точкой без цифр после) НЕ является float — лексер не распознаёт такую запись. Всегда пишите хотя бы один знак после точки: `1.0`

### Строки (string)

```tiger
var name = "Иван"
var greeting = 'Привет!'   # одинарные кавычки тоже работают
var empty = ""
```

Экранирование внутри строк:

| Последовательность | Символ |
|--------------------|--------|
| `\n` | Новая строка |
| `\t` | Табуляция |
| `\r` | Возврат каретки |
| `\\` | Обратный слеш |
| `\"` | Двойная кавычка |
| `\'` | Одинарная кавычка |

```tiger
var msg = "Строка 1\nСтрока 2"
var path = "C:\\Users\\Ivan"
var quote = "Он сказал: \"привет\""
```

### Булевы значения (bool)

```tiger
var flag = true
var done = false
```

Ключевые слова: `true` и `false` (строчные).

### Null

```tiger
var result = null
```

Используется как отсутствие значения. В C++ транспилируется в `nullptr`.

> **Нюанс:** `null` нельзя сравнивать со строками или числами — используйте его только для проверки `== null`.

## 6. Переменные

### Объявление

Переменные объявляются через ключевое слово `var`:

```tiger
# Без типа — автоматический вывод
var x = 10
var name = "Tiger"
var active = true

# С явным типом — статическая типизация
var count: int = 10
var price: float = 99.99
var message: string = "Hello"
var flag: bool = true
var items: list = [1, 2, 3]
```

### Переприсвоение

После объявления `var` не нужен:

```tiger
var score = 0
score = 100
score = score + 50
```

### Составные операторы присваивания

```tiger
var n = 10
n += 5    # n = n + 5   → 15
n -= 3    # n = n - 3   → 12
n *= 2    # n = n * 2   → 24
n /= 4    # n = n / 4   → 6
```

### Область видимости

- Переменная, объявленная внутри функции — **локальная**, видна только в этой функции.
- Переменная, объявленная **вне всех функций** — **глобальная**, видна везде.

```tiger
var global_count = 0   # глобальная — видна в increment() и в main()

increment() {
    global_count += 1  # изменяем глобальную переменную
}

main() {
    increment()
    increment()
    print(str.to_str(global_count))   # выведет: 2
}
```

## 7. Операторы

### Арифметические

| Оператор | Описание | Пример |
|----------|----------|--------|
| `+` | Сложение / конкатенация строк | `5 + 3` → `8` |
| `-` | Вычитание / унарный минус | `10 - 4` → `6` |
| `*` | Умножение | `3 * 4` → `12` |
| `/` | Деление | `10 / 4` → `2.5` |
| `%` | Остаток от деления | `10 % 3` → `1` |

### Сравнения

| Оператор | Описание |
|----------|----------|
| `==` | Равно |
| `!=` | Не равно |
| `<` | Меньше |
| `<=` | Меньше или равно |
| `>` | Больше |
| `>=` | Больше или равно |

```tiger
var a = 5
var b = 10
print(a < b)    # true
print(a == 5)   # true
print(a != b)   # true
```

### Логические

| Оператор | Описание | Пример |
|----------|----------|--------|
| `and` | Логическое И | `true and false` → `false` |
| `or` | Логическое ИЛИ | `true or false` → `true` |
| `not` | Логическое НЕ | `not true` → `false` |

```tiger
var age = 25
var has_id = true

if (age >= 18 and has_id) {
    print("Доступ разрешён")
}
```

### Конкатенация строк

Оператор `+` соединяет строки. Чтобы добавить число, его нужно сначала преобразовать:

```tiger
var score = 42
print("Счёт: " + str.to_str(score))   # правильно
# print("Счёт: " + score)             # ошибка компиляции!
```

### Приоритет операторов (от высшего к низшему)

```
not  (унарный)
* / %
+ -
< <= > >=
== !=
and
or
```

## 8. Строки и str.*

Модуль `str` содержит встроенные функции для работы со строками. Вызываются через точку: `str.функция(аргументы)`.

### str.len(s) — длина строки

```tiger
var s = "Привет"
var n = str.len(s)
print(str.to_str(n))   # 6
```

### str.to_str(x) — число → строка

```tiger
var pi = 3.14
var text = "Пи равно " + str.to_str(pi)
print(text)   # Пи равно 3.140000
```

Работает как с `int`, так и с `float`.

### str.to_int(s) — строка → целое число

```tiger
var s = "42"
var n = str.to_int(s)
print(str.to_str(n + 8))   # 50
```

> **Нюанс:** Если строка не является корректным числом, программа завершится с ошибкой во время выполнения. Проверяйте ввод пользователя перед конвертацией.

### str.to_float(s) — строка → дробное число

```tiger
var s = input("Введите число: ")
var x = str.to_float(s)
print("Удвоенное: " + str.to_str(x * 2.0))
```

### str.upper(s) — в верхний регистр

```tiger
var s = "hello"
print(str.upper(s))   # HELLO
```

> **Нюанс:** Работает корректно только для латинских букв (ASCII). Кириллица не конвертируется — ограничение C++ `std::toupper`.

### str.lower(s) — в нижний регистр

```tiger
var s = "WORLD"
print(str.lower(s))   # world
```

> Те же ограничения по кириллице, что и у `str.upper`.

### Полная таблица str.*

| Функция | Аргументы | Возвращает | Пример |
|---------|-----------|------------|--------|
| `str.len(s)` | строка | int | `str.len("hi")` → `2` |
| `str.to_str(x)` | int или float | строка | `str.to_str(3.14)` → `"3.140000"` |
| `str.to_int(s)` | строка | int | `str.to_int("5")` → `5` |
| `str.to_float(s)` | строка | float | `str.to_float("1.5")` → `1.5` |
| `str.upper(s)` | строка | строка | `str.upper("hi")` → `"HI"` |
| `str.lower(s)` | строка | строка | `str.lower("HI")` → `"hi"` |
| `str.trim(s)` | строка | строка | `str.trim("  hi  ")` → `"hi"` |
| `str.split(s)` | строка | список строк | `str.split("a,b,c")` → `["a", "b", "c"]` |
| `str.startswith(s, prefix)` | строка, префикс | bool | `str.startswith("hello", "he")` → `true` |
| `str.endswith(s, suffix)` | строка, суффикс | bool | `str.endswith("hello", "lo")` → `true` |

## 9. Ввод и вывод

### print() — вывод

```tiger
print("Привет, мир!")          # строка
print(42)                       # число
print(3.14)                     # дробное
print(true)                     # булево
print()                         # пустая строка (просто перенос строки)
```

Несколько аргументов разделяются пробелом:

```tiger
var name = "Иван"
var age = 30
print("Имя:", name, "Возраст:", age)
# → Имя: Иван Возраст: 30
```

Для вывода строки и числа слитно используйте конкатенацию:

```tiger
var n = 100
print("Результат: " + str.to_str(n))   # Результат: 100
```

> **Нюанс:** `print` всегда добавляет перевод строки в конце. Вывод без переноса — невозможен в текущей версии языка.

### input() — ввод с клавиатуры

```tiger
var name = input("Как вас зовут? ")
print("Привет, " + name + "!")
```

Без аргумента — читает без подсказки:

```tiger
var line = input()
```

Важно: `input()` **всегда возвращает строку**. Для работы с числами используйте конвертацию:

```tiger
var age_str = input("Ваш возраст: ")
var age = str.to_int(age_str)

if (age >= 18) {
    print("Совершеннолетний")
} else {
    print("Несовершеннолетний")
}
```

## 10. Условия: if / elif / else

### Базовая форма

```tiger
if (условие) {
    # тело
}
```

### С else

```tiger
var x = 10

if (x > 0) {
    print("Положительное")
} else {
    print("Ноль или отрицательное")
}
```

### С elif

```tiger
var score = 75

if (score >= 90) {
    print("Отлично")
} elif (score >= 70) {
    print("Хорошо")
} elif (score >= 50) {
    print("Удовлетворительно")
} else {
    print("Неудовлетворительно")
}
```

Количество `elif` не ограничено.

### Вложенные условия

```tiger
var x = 5
var y = 10

if (x > 0) {
    if (y > 0) {
        print("Оба положительные")
    } else {
        print("x положительный, y — нет")
    }
} else {
    print("x не положительный")
}
```

### Важные правила

1. Скобки вокруг условия **обязательны**: `if (x > 0)` — правильно, `if x > 0` — ошибка парсера.
2. Фигурные скобки `{}` обязательны всегда, даже для одного оператора.
3. `elif` и `else` должны идти **сразу после** закрывающей `}` предыдущего блока.

```tiger
# ПРАВИЛЬНО:
if (x > 0) {
    print("да")
} elif (x == 0) {
    print("ноль")
} else {
    print("нет")
}

# НЕПРАВИЛЬНО — else на отдельной строке после пустой строки
if (x > 0) {
    print("да")
}

else {            # ← парсер не найдёт связь с if
    print("нет")
}
```

## 11. Циклы: while

Единственный тип цикла в Tiger — `while`.

### Базовая форма

```tiger
var i = 0
while (i < 5) {
    print(str.to_str(i))
    i += 1
}
# выведет: 0, 1, 2, 3, 4
```

### Бесконечный цикл с выходом через флаг

```tiger
var running = true

while (running) {
    var choice = input("Введите 'q' для выхода: ")
    if (choice == "q") {
        running = false
    } else {
        print("Вы ввели: " + choice)
    }
}
```

### Цикл со счётчиком вниз

```tiger
var n = 10
while (n > 0) {
    print(str.to_str(n))
    n -= 1
}
print("Пуск!")
```

### Вложенные циклы

```tiger
var i = 1
while (i <= 3) {
    var j = 1
    while (j <= 3) {
        print(str.to_str(i) + " x " + str.to_str(j) + " = " + str.to_str(i * j))
        j += 1
    }
    i += 1
}
```


> **Нюанс:** В Tiger нет `break` и `continue`. Для выхода из цикла используйте флаг-переменную `running = false` или `return` внутри функции.

## 12. Циклы: for

for-цикл позволяет перебирать элементы списка или словаря.

### Синтаксис

```tiger
for (item in collection) {
    # тело цикла
}
```

### Примеры

```tiger
# Перебор списка
var nums = [1, 2, 3, 4, 5]
for (n in nums) {
    print(str.to_str(n))
}
# выведет: 1 2 3 4 5

# Перебор строки посимвольно
var text = "hello"
for (ch in text) {
    print(ch)
}
```

### Ограничения

- Цикл `for` работает с: списками, строками, вызовами функций (возвращающих список/строку)
- Для словарей можно использовать `for (key in dict)` - будет перебор по ключам

## 13. Функции

### Объявление

Функции могут иметь явные типы параметров и возвращаемого значения:

```tiger
# Без типов (динамическая типизация)
greet(name) {
    print("Привет, " + name + "!")
}

# С типами (статическая типизация)
add(a: int, b: int): int {
    return a + b
}

multiply(x: float, y: float): float {
    return x * y
}

getName(): string {
    return "Tiger"
}
```

### Вызов

```tiger
var result = add(3, 4)      # result = 7
greet("Мир")                # выведет: Привет, Мир!
print(str.to_str(multiply(2.5, 4.0)))  # 10
```

### Вызов

```tiger
var result = add(3, 4)      # result = 7
greet("Мир")                # выведет: Привет, Мир!
```

### Возвращаемое значение

```tiger
max(a, b) {
    if (a > b) {
        return a
    }
    return b
}

main() {
    var m = max(10, 25)
    print(str.to_str(m))   # 25
}
```

Функция без `return` возвращает ничего (void в C++). Вызывать её как выражение нельзя.

### Функция без параметров

```tiger
say_hello() {
    print("Привет!")
}

main() {
    say_hello()
}
```

### Рекурсия

```tiger
factorial(n) {
    if (n <= 1) {
        return 1
    }
    return n * factorial(n - 1)
}

main() {
    print(str.to_str(factorial(6)))   # 720
}
```

### Важные правила

1. Функции нельзя объявлять внутри других функций (вложенные функции запрещены).
2. Параметры функции не имеют типа — они принимают любое значение.
3. Функция должна быть **объявлена до того, как она вызывается**. Порядок в файле важен: вспомогательные функции — выше `main()`.
4. `main()` — особая функция. Транспилятор превращает её в `int main()` C++.

## 14. Списки

Список — упорядоченная коллекция элементов одного типа.

### Создание

```tiger
var nums = [1, 2, 3, 4, 5]
var words = ["яблоко", "банан", "вишня"]
var empty = []
var floats = [1.0, 2.5, 3.7]
```

### Доступ по индексу

Индексы начинаются с `0`:

```tiger
var fruits = ["яблоко", "банан", "вишня"]
print(fruits[0])   # яблоко
print(fruits[2])   # вишня
```

### Изменение элемента

```tiger
var nums = [10, 20, 30]
nums[1] = 99
print(str.to_str(nums[1]))   # 99
```

### Длина списка

```tiger
var nums = [1, 2, 3, 4, 5]
var n = str.len(str.to_str(nums[0]))   # не для списков!
```

> **Нюанс:** У списков нет встроенного метода `.len()`. Для определения длины нужно хранить её отдельной переменной или вычислять вручную.

### Перебор списка

```tiger
var nums = [10, 20, 30, 40, 50]
var i = 0
var total = 0
while (i < 5) {          # 5 — длина, заданная вручную
    total += nums[i]
    i += 1
}
print("Сумма: " + str.to_str(total))   # 150
```

### Списки как параметры

```tiger
sum_list(lst, size) {
    var total = 0
    var i = 0
    while (i < size) {
        total += lst[i]
        i += 1
    }
    return total
}

main() {
    var nums = [1, 2, 3, 4, 5]
    var s = sum_list(nums, 5)
    print(str.to_str(s))   # 15
}
```

### Ограничения списков

- Все элементы должны быть одного типа (точнее — C++ должен выводить `std::common_type`).
- Список строк и список чисел нельзя смешивать: `[1, "hello"]` — ошибка компиляции.
- Нет встроенных методов `.push()`, `.pop()`, `.append()` — в текущей версии списки фиксированного размера.

## 14.1 Словари (dict)
 
Словарь — коллекция пар ключ-значение. Ключи должны быть строками, значения могут быть любого типа.
 
 ### Создание
 
 ```tiger
 var config = {"port": 8080, "timeout": 30}
 var empty = {}
 ```
 
 > **Нюанс:** В текущей версии значения словаря должны быть числами (int/float). Строковые значения не поддерживаются — словарь транслируется в `std::map<std::string, double>`.
 
 ### Доступ к элементам
 
 ```tiger
 var config = {"host": "localhost", "port": 8080}
 print(config["host"])   # localhost
 print(config["port"])   # 8080
 ```
 
### Автоматическое изменение и добавление элементов
  ```tiger
  var counts = {"zen1": 0, "zen2": 0}
  counts["zen1"] = 1      # изменить существующий ключ
  counts["zen3"] = 2      # добавить новый ключ
  ```
 
### Ограничения словарей
  - Ключи должны быть строками
  - Значения должны быть числами (float/double) — строковые значения не поддерживаются
  - В C++ транслируется в `std::map<std::string, double>`
  - Вложенные словари не поддерживаются — значение должно быть числом

 ## 15. Глобальные переменные

Переменная, объявленная **вне любой функции**, является глобальной. Она видна во всех функциях файла.

### Объявление

```tiger
var counter = 0          # глобальная int
var app_name = "Tiger"   # глобальная string
var pi = 3.14159         # глобальная double
var debug = false        # глобальная bool
```

### Чтение и изменение из функций

```tiger
var score = 0

add_score(points) {
    score += points   # изменяем глобальную переменную напрямую
}

reset_score() {
    score = 0
}

main() {
    add_score(10)
    add_score(25)
    print("Счёт: " + str.to_str(score))   # 35
    reset_score()
    print("После сброса: " + str.to_str(score))   # 0
}
```

### Как транспилируются глобальные переменные

Tiger превращает топ-уровневые `var` в настоящие C++ глобальные переменные:

```tiger
var x = 42
var name = "Tiger"
```

→ генерируется в C++ как:

```cpp
int x = 42;
std::string name = std::string("Tiger");
```

Эти переменные живут всё время работы программы.

### Таблица типов глобальных переменных

| Tiger | C++ тип |
|-------|---------|
| `var x = 42` | `int` |
| `var x = 3.14` | `double` |
| `var x = "hello"` | `std::string` |
| `var x = true` | `bool` |
| `var x = null` | `void*` |
| `var x = [1,2,3]` | `std::vector<double>` |
| `var x` (без значения) | `double` |

> **Нюанс:** Глобальная переменная фиксирует тип при объявлении. Нельзя объявить `var x = 0` (int) и потом присвоить `x = "hello"` (string) — это ошибка компиляции C++.

> **Нюанс:** Если глобальная переменная инициализируется сложным выражением (вызов функции, арифметика), тип автоматически выводится как `double`. Это безопасный дефолт для числовых вычислений.

### Импорт .tgr файлов

```tiger

```

Функции из `utils.tgr` становятся доступны в вашем файле.

**Правила импорта .tgr:**
- Файл ищется сначала в папке текущего файла, затем в папках из `-I`.
- Функция `main()` в импортируемом файле **игнорируется** — вы подключаете только вспомогательные функции.
- Циклические импорты (`A` импортирует `B`, `B` импортирует `A`) **запрещены** — транспилятор выдаст ошибку.
- Каждый файл импортируется только один раз, даже если несколько файлов ссылаются на него.

## 17. Важные нюансы и ограничения

### Статическая типизация (v0.3+)

Tiger поддерживает статическую типизацию — проверка типов на этапе компиляции.

#### Преимущества:
- Ошибки типов находятся до запуска программы
- Явные типы делают код самодокументируемым  
- Генерируется более эффективный C++ код

#### Синтаксис типизации:

```tiger
# Переменные
var x: int = 10
var name: string = "Tiger"

# Функции
add(a: int, b: int): int {
    return a + b
}

# Без указания типов — динамическая типизация
var y = 20
```

#### Ошибки типизации:

```tiger
var x: int = "hello"   # ОШИБКА: cannot assign string to int
```

### Нет break и continue

Для выхода из цикла — флаг:

```tiger
var found = false
var i = 0
while (i < 10 and not found) {
    if (nums[i] == target) {
        found = true
    }
    i += 1
}
```

### Нет вложенных функций

```tiger
outer() {
    inner() {    # ← ОШИБКА: вложенные функции запрещены
        print("вложенная")
    }
}
```

### Нет try/catch

Обработки исключений нет. Если `str.to_int("abc")` — программа упадёт с ошибкой C++.

### print всегда добавляет \n

Нельзя вывести текст без перевода строки.

### Порядок функций важен

Функция должна быть объявлена выше места вызова. Нет автоматических forward declarations.

```tiger
# ПРАВИЛЬНО:
helper() {
    print("помощник")
}

main() {
    helper()   # helper объявлена выше
}

# НЕПРАВИЛЬНО:
main() {
    helper()   # helper объявлена НИЖЕ — ошибка компиляции
}

helper() {
    print("помощник")
}
```
